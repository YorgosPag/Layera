### 📊 Πλήρης Έλεγχος Όλων των Προβλημάτων

```bash
npm run verify
```

**📜 Scripts που ενεργοποιούνται:**
- `npm run typecheck` → TypeScript validation across all workspaces
- `npm run lint` → ESLint validation across all workspaces
- `npm run policy:check` → Pre-commit hooks validation
- `npm run dup:check` → jscpd duplicate detection tool
- `npm run enterprise:validate` → `scripts/domains/validation/validate-duplicates.js`

**Τι κάνει με απλά λόγια:**
Αυτή η εντολή είναι σαν να κάνεις έναν πλήρη τεχνικό έλεγχο στο αυτοκίνητό σου. Ελέγχει ΟΛΕΣ τις πτυχές του κώδικα σου - από τα τεχνικά σφάλματα μέχρι τα προβλήματα στυλ, από τα διπλότυπα μέχρι τις πολιτικές της εταιρείας. Σαν να έχεις έναν επιθεωρητή που ελέγχει κάθε γωνία του σπιτιού σου για να βεβαιωθεί ότι όλα είναι τέλεια.

**Χρονική διάρκεια:** 5-10 λεπτά
**Τι θα δεις:** Μια πλήρη αναφορά με όλα τα προβλήματα που βρέθηκαν στον κώδικα

**⚠️ ΚΙΝΔΥΝΟΙ:**
- Κανένας κίνδυνος - αυτή η εντολή μόνο διαβάζει και ελέγχει, δεν αλλάζει τίποτα
- Μπορεί να πάρει πολύ χρόνο αν έχεις μεγάλο έργο
- Μπορεί να σου δώσει χιλιάδες μηνύματα σφαλμάτων που μπορεί να σε μπερδέψουν

**Τι να κάνεις πριν τρέξεις την εντολή:**
1. Βεβαιώσου ότι έχεις κλείσει όλα τα άλλα προγράμματα που τρέχουν
2. Έχε έτοιμο ένα φλιτζάνι καφέ γιατί θα πάρει λίγο χρόνο
3. Μην ανησυχείς αν δεις πολλά μηνύματα - είναι φυσιολογικό

### 🚨 Κρίσιμοι Έλεγχοι Ταχείας Επεξεργασίας

```bash
npm run validation:critical
```

**📜 Script που ενεργοποιείται:**
- `scripts/domains/validation/critical-validation.js`

**Τι κάνει με απλά λόγια:**
Σκέψου το σαν έναν γιατρό επειγόντων περιστατικών. Αντί να κάνει πλήρη εξέταση, ψάχνει μόνο για τα πιο σοβαρά προβλήματα που θα μπορούσαν να "σκοτώσουν" την εφαρμογή σου. Είναι γρήγορος και στοχευμένος - σαν να ελέγχεις αν έχει κλείσει η πόρτα του σπιτιού πριν φύγεις για διακοπές.

**Χρονική διάρκεια:** 30 δευτερόλεπτα
**Τι θα δεις:** Μόνο τα πιο σοβαρά προβλήματα που χρειάζονται άμεση επίλυση

**⚠️ ΚΙΝΔΥΝΟΙ:**
- Κανένας κίνδυνος - μόνο διαβάζει τον κώδικα
- Μπορεί να μην σου δείξει μικρότερα προβλήματα που όμως αξίζουν προσοχή

**Τι να κάνεις πριν τρέξεις την εντολή:**
1. Αυτό είναι η πιο ασφαλής εντολή - μπορείς να την τρέχεις όποτε θέλεις
2. Ιδανική για καθημερινή χρήση πριν αρχίσεις δουλειά

### 🔄 Εντοπισμός Διπλότυπου Κώδικα

```bash
npm run dup:check
```

**📜 Script που ενεργοποιείται:**
- `jscpd` (external tool) με παραμέτρους: `--pattern "**/*.{ts,tsx,js,jsx,css,scss}" -t 0 --ignore "**/dist/**,**/node_modules/**,packages/tokens/**,packages/styles/**,**/*.svg"`

**Τι κάνει με απλά λόγια:**
Φανταστείσου ότι έχεις ένα μεγάλο σπίτι και θέλεις να βρεις αν έχεις αγοράσει τα ίδια έπιπλα δύο φορές. Αυτή η εντολή ψάχνει στον κώδικά σου για κομμάτια που είναι πανομοιότυπα ή πολύ παρόμοια. Είναι σαν να έχεις έναν ντετέκτιβ που βρίσκει τα "δίδυμα" κομμάτια κώδικα που κάνουν την ίδια δουλειά.

**Χρονική διάρκεια:** 2-3 λεπτά
**Τι θα δεις:** Λίστα με κομμάτια κώδικα που επαναλαμβάνονται

**⚠️ ΚΙΝΔΥΝΟΙ:**
- Κανένας κίνδυνος - μόνο αναζητά και αναφέρει
- Μπορεί να σου δώσει πολλά αποτελέσματα που να χρειάζονται χρόνο για ανάλυση

**Τι να κάνεις πριν τρέξεις την εντολή:**
1. Κανένας ειδικός προετοιμασία - είναι απόλυτα ασφαλής
2. Καλό είναι να έχεις χρόνο να διαβάσεις τα αποτελέσματα

**Τι να κάνεις πριν τρέξεις την εντολή:**
1. Αυτή είναι η πιο φιλική εντολή για αρχάριους

### 🔷 Έλεγχοι Σφαλμάτων TypeScript

```bash
npm run typecheck
```

**📜 Scripts που ενεργοποιούνται:**
- `npm run typecheck --workspaces --if-present` → TypeScript compiler check σε όλα τα packages

**Τι κάνει με απλά λόγια:**
Το TypeScript είναι μια γλώσσα προγραμματισμού που είναι πολύ αυστηρή με τους κανόνες. Σκέψου το σαν έναν αυστηρό καθηγητή γραμματικής που ελέγχει κάθε πρόταση που γράφεις για να βεβαιωθεί ότι είναι γραμματικά σωστή. Αυτή η εντολή ελέγχει αν όλος ο κώδικας σου ακολουθεί τους κανόνες της γλώσσας.

**Τι να κάνεις πριν τρέξεις την εντολή:**
1. Βεβαιώσου ότι έχεις κλείσει άλλα προγράμματα γιατί χρησιμοποιεί πολλή μνήμη
2. Προετοιμάσου να δεις πολλά τεχνικά μηνύματα
3. Μην πανικοβληθείς αν δεις εκατοντάδες σφάλματα - είναι συνηθισμένο

### 🔗 Έλεγχοι Εξαρτήσεων και Εισαγωγών

```bash
npm run imports:check
```

**Τι κάνει με απλά λόγια:**
Στον προγραμματισμό, τα διαφορετικά μέρη του κώδικα "μιλούν" μεταξύ τους. Αυτή η εντολή είναι σαν έναν ψυχολόγο οικογένειας που ελέγχει αν όλα τα μέλη της οικογένειας επικοινωνούν υγιή μεταξύ τους και δεν υπάρχουν τοξικές σχέσεις ή κυκλικές εξαρτήσεις (σαν το Α να εξαρτάται από το Β, το Β από το Γ, και το Γ από το Α).

**Χρονική διάρκεια:** 1 λεπτό
**Τι θα δεις:** Προβλήματα στις σχέσεις μεταξύ των μερών του κώδικα

**⚠️ ΚΙΝΔΥΝΟΙ:**
- Κανένας κίνδυνος - μόνο αναλύει τις σχέσεις του κώδικα
- Τα μηνύματα μπορεί να είναι πολύ τεχνικά για αρχάριους

**Τι να κάνεις πριν τρέξεις την εντολή:**
1. Τίποτα ειδικό - είναι ασφαλής έλεγχος

### 🚀 CI/CD Pipeline για Naming Standards

**Τι κάνει με απλά λόγια:**
Αυτό είναι ένα αυτόματο σύστημα που ελέγχει κάθε αλλαγή στον κώδικα σου για naming compliance πριν επιτρέψει το deployment. Σαν να έχεις security guard στην είσοδο της εταιρείας που ελέγχει αν όλοι φορούν τη σωστή στολή πριν τους αφήσει να μπουν.

**Πώς ενεργοποιείται:**
- Αυτόματα σε κάθε git push στο main/develop
- Αυτόματα σε κάθε Pull Request
- Manual trigger μέσω GitHub Actions

**Τι κάνει το CI/CD pipeline:**

#### 🔍 **Αυτόματος Έλεγχος (σε κάθε push/PR):**
```bash
# Αυτόματα ενεργοποιείται:
.github/workflows/naming-conventions-check.yml
```

**Stages του Pipeline:**
1. **Naming Validation** - Τρέχει τον έλεγχο naming conventions
2. **Terminology Check** - Ελέγχει συνέπεια ορολογίας
3. **Auto-fix** (conditional) - Αν είναι ασφαλές, κάνει auto-fixes
4. **Compliance Gate** - Αποκρούει deployment αν score < 85%
5. **Reporting** - Δημιουργεί compliance reports

#### 🎯 **Compliance Levels:**
- **🥇 GOLD (95%+)**: Excellent enterprise standard
- **🥈 SILVER (85%+)**: Good enterprise compliance - **PASS**
- **🥉 BRONZE (75%+)**: Minimum acceptable
- **❌ FAILED (<75%)**: Unacceptable for enterprise - **BLOCKED**

#### 🤖 **Manual Trigger με Auto-fix:**
```bash
gh workflow run "Enterprise Naming Conventions" --ref=main -f auto_fix=true
```

**⚠️ CI/CD ΚΙΝΔΥΝΟΙ:**
- **ΜΕΣΑΙΟΣ ΚΙΝΔΥΝΟΣ**: Auto-fix στο CI μπορεί να κάνει αλλαγές στο branch σου
- Μπορεί να block deployments αν δεν πετύχεις 85% compliance
- Critical violations (CRITICAL severity) μπορούν να block ακόμη και με καλό score
- Το auto-fix τρέχει μόνο για PR branches, όχι για main

**Τι να κάνεις για CI/CD:**
1. **Τρέξε local validation πρώτα**: `npm run naming:validate`
2. **Fix issues τοπικά πρώτα**: `npm run naming:fix-preview` και μετά `npm run naming:fix`
3. **Commit clean code**: Το CI πρέπει να βρίσκει compliance ≥85%
4. **Monitor το CI**: Αν fail, κοίτα τα CI logs για λεπτομέρειες
5. **Use auto-fix wisely**: Μόνο για minor violations, όχι για major refactoring

#### 📊 **Compliance Monitoring:**
Το CI/CD σύστημα δημιουργεί:
- **PR Comments** με compliance status
- **GitHub Labels** (naming-compliance-gold/silver/bronze/failed)
- **Artifacts** με detailed reports
- **Branch protection** rules based on compliance
